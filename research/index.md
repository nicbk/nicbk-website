# Research

This directory holds research artifacts that inform planning decisions for
the project. Each category is a folder (so concerns within it can be split
into their own file or subfolder as they grow in complexity — see
[AGENTS.md](../AGENTS.md)), written from web-search findings (per
[AGENTS.md](../AGENTS.md) — research relies on searching the web, not
built-in knowledge), and topics within a category are refined and researched
incrementally.

## Categories

- [technologies/index.md](./technologies/index.md) — What technologies/services
  to use (stack, third-party tools/APIs).
- [ui-ux/index.md](./ui-ux/index.md) — Design and interaction patterns.
- [system-architecture/index.md](./system-architecture/index.md) — How shared
  infrastructure and sub-applications fit together.
- [coding-conventions/index.md](./coding-conventions/index.md) — Language/style
  conventions and code organization.
- [project-management-conventions/index.md](./project-management-conventions/index.md) —
  How work is tracked, planned, and communicated.
- [security-privacy/index.md](./security-privacy/index.md) — Auth, data
  protection, and encryption practices.
- [testing-qa/index.md](./testing-qa/index.md) — What/how to test, and with
  what tooling.
- [devops-deployment/index.md](./devops-deployment/index.md) — Build, deploy,
  hosting, and CI practices.
- [data-modeling/index.md](./data-modeling/index.md) — Schema/shape design
  for shared reactive data.
- [accessibility/index.md](./accessibility/index.md) — Baseline accessibility
  standard.
- [licensing/index.md](./licensing/index.md) — License compatibility/attribution
  given the open-source-only constraint.
- [documentation-content-conventions/index.md](./documentation-content-conventions/index.md) —
  Conventions for docs and MDX blog content.
